How the Quality Score is Computed

The Quality Score is a normalized metric (0 - 100%) that reflects the overall data quality for a source. It is displayed as a percentage badge in the table.

Components typically considered include:

  • Completeness how many required fields are present.
  • Uniqueness degree of duplicate or identical records.
  • Freshness how up-to-date the data is.
  • Schema conformity how well records match expected types/format.

Internally the score is calculated as a weighted sum of these components and stored as a fraction (0.0 - 1.0) in the source metadata. The dashboard displays the value as a percentage (for example, 0.87 87%).

Note: this is an aggregate indicator. For diagnosing specific issues, inspect source previews and per-field validation reports in the data preview tools.
