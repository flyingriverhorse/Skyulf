## v0.5.4 — Type-Check Stabilization (branch `054`)

> Branch `054` is a focused type-check cleanup: cast refactors in `DataSplitter`, `# ty: ignore` placement on optional-dependency imports, and consistent import ordering. No runtime behaviour changes.

### 🐛 Bug Fixes
- **ty type-check: 145 → 0 diagnostics** (post-`black`-reformat regression). Five-pronged cleanup so CI runs green:
  - **Series[Any] cascade (~80 errors):** `unpack_pipeline_input` was declared `-> Tuple[Union[pd.DataFrame, SkyulfDataFrame], ...]`, but `ty` narrowed `data[0]` of the tuple-input branch to `Series[Any]` (since `pd.DataFrame[col]` returns Series). The poisoned union propagated into every preprocessing module's `X.with_columns(...)` / `X.to_pandas()` call, producing dozens of `call-non-callable` errors. Return type relaxed to `Tuple[Any, Optional[Any], bool]`.
  - **Optional dependencies:** Added `# ty: ignore[unresolved-import]` to ~20 try/except-guarded imports of `s3fs`, `redis`, `xgboost`, `optuna`, `optuna.integration*`, `rich.*`, `matplotlib.pyplot`, `vaderSentiment`, `causallearn`, `rapidfuzz` across `backend/`, `run_fastapi.py`, `skyulf-core/`. Set `unused-ignore-comment = "ignore"` and `redundant-cast = "ignore"` in `[tool.ty.rules]` so the same suppressions work whether or not the optional package is installed locally.
  - **`backend/data_ingestion/serialization.py`:** Annotated the `metadata` dict literal as `Dict[str, Any]` so `metadata["sample_data"] = sample_data` no longer fails the inferred narrow value-type union.
  - **`skyulf-core/skyulf/modeling/base.py`:** Added `cast(Any, X_test)` (and `X_val`) immediately after unpacking the tuple — `isinstance(dataset.test, tuple)` + `hasattr(X_test, "columns")` had narrowed it to a `<Protocol with members 'columns'>` that lacks `.drop`.
  - **Misc:** `pl.from_pandas(cast(pd.DataFrame, df_out))` in `transformations.py`; `np.asarray(series.values).reshape(-1, 1)` in `bucketing.py` (so ExtensionArray inputs go through ndarray); `splitter.split_xy(cast(Any, X), y)` in `split.py`; `y_res: Any` annotation in `pipeline._metrics_resampling`.
- **Black/ty import-wrapping interaction:** Black wrapped long `from x import Y  # ty: ignore[unresolved-import]` statements into multi-line parenthesized form, which moved the comment to the closing-paren line where `ty` no longer applied it. Moved `# ty: ignore[unresolved-import]` onto the `from ... import (` line itself in `tuning/engine.py` (3 sites) and `profiling/analyzer.py` (1 site).
- **`DataSplitter` Series[Any] revival:** Replaced bare `: Any` annotations on `X_any`/`df_any`/`data_pl` in `preprocessing/split.py` polars branches with `cast(Any, ...)` — the annotation alone was not enough to break union narrowing on `.to_pandas()` / `.to_series()`, but the cast returns a fresh `Any`-typed value that does.

## v0.5.3 — Per-Path Data Preview & Branch Coloring (branch `053`)

### ✨ Features
- **Per-Path Tabs in Data Preview Node:** Connecting two or more upstream paths into a `Data Preview` node now renders each path in its own tab (mirroring the parallel-execution Results panel) instead of silently column-merging them. Tabs are labeled `Path A · <source node>`, `Path B · <source node>`, …, and matching path colors propagate to the canvas edges via `useBranchColors`. Backend auto-parallelizes any `data_preview` terminal with 2+ inputs (no `execution_mode` flag needed) by adding `AUTO_PARALLEL_STEP_TYPES` in `graph_utils.py`. Frontend `DataPreviewSettings` polls all per-branch jobs (new `lastRunJobIds` field, legacy `lastRunJobId` preserved) and exposes a branch tab bar above the existing train/test/validation split tabs.
- **UI propagation for `data_preview`:** `CustomNodeWrapper` shows the parallel (GitFork) badge instead of the merge badge on a multi-input `data_preview`; `PropertiesPanel` hides the Merge Strategy dropdown for `data_preview` (irrelevant when each path is shown separately); `useGraphStore.onConnect` no longer surfaces the auto-merge confirm dialog when wiring into a `data_preview`.
- **Per-terminal Path letters:** `useBranchColors` now restarts path letters per terminal so canvas edge labels (`Path A · Encoding`, `Path B · Encoding`) line up with the per-terminal tab bars instead of running through a single global counter (which previously produced labels like `Path E / Path F` on the canvas while the tabs read `Path A / Path B`).
- **Preview shows all columns:** Preview table renders **all columns** (previously hard-capped at 5, hiding columns added by transformers like `MissingIndicator`) with horizontal scroll and a sticky header showing the full row/column count alongside the visible-row sample size.

### 🐛 Bug Fixes
- **Run All Experiments no longer queues Data Preview jobs:** `Toolbar.handleRunAll` strips `data_preview` nodes (and their edges) before sending the pipeline to `/pipeline/run`. Backend treats `data_preview` as a terminal in `partition_parallel_pipeline`, so leaving it in spawned a bogus `preview` job alongside the real training runs. Preview is an inspection-only sink invoked exclusively from its own "Run Preview" button.

## v0.5.1 — Multi-Branch Preview, Merge Engine & CI Tooling (branch `051`)

### 🐛 Bug Fixes
- **CI `ty` failure on missing `.venv`:** Removed `python = ".venv"` from `[tool.ty.environment]` in `pyproject.toml` — it pointed at a local-only path that broke CI runs (`Invalid setting in configuration file`). `ty` now discovers the active interpreter automatically.
- **CI `ty` scope alignment:** Dropped top-level `tests/` from the CI `ty` command and from `[tool.ty.src].include` to match the documented local command in `quality_checks.instructions.md`.
- **Lint noise reduction:** Added `site,docs/examples/scripts,build,dist` to `.flake8` `exclude`. `site/` is generated mkdocs output; `docs/examples/scripts` are illustrative snippets, neither belongs under production lint scope.
- **`POST /api/pipeline/run` 500 on granular step types:** `run_pipeline` and `preview` strictly coerced incoming `step_type` via `StepType(...)`, which only accepts the four coarse categories. Frontend emits fine-grained kinds (`DropMissingColumns`, `StandardScaler`, …) already handled by the engine's generic transformer fallback. Added `coerce_step_type()` helper in `backend/ml_pipeline/execution/schemas.py`, widened `NodeConfig.step_type` to `Any`, and swapped both call sites in `backend/ml_pipeline/api.py`.
- **Merge Everywhere:** Multi-input merging now applies to *all* node types (preview, transformers, feature engineering), not just training/tuning. Previously, a node with two parents silently kept only the first input — so connecting two preprocessing branches into a Preview, Deduplicate, etc. dropped one branch entirely.
- **SplitDataset Preserved Through Merge:** When all inputs to a downstream node are `SplitDataset`s, the merge now produces a `SplitDataset` keeping `train`, `test`, and `validation` halves instead of flattening to train-only. Fixes silently missing `test_*` metrics when training merged after a Splitter → parallel preprocessing fan-out.
- **Index-Aware Column Merge:** `_merge_frames` now resets indices before column-wise concatenation, preventing NaN rows when upstream branches kept different row indices.
- **Last-Wins Column Merge (Bug 9i):** `_merge_frames` now uses last-wins semantics on overlapping columns instead of first-wins. A scaler placed *after* a splitter feeding back into a downstream node now actually contributes its scaled columns rather than being silently dropped because the unscaled sibling appeared first in the input list.
- **(X, y) Tuple Preserved Through Splitter (Bug 9c):** `SplitApplier.apply` now detects `(X, y)` tuple inputs and a `target_column` present in plain DataFrame inputs, and emits clean `SplitDataset` slots holding `(X, y)` tuples per split. Downstream transformers no longer see `(df, None)` placeholders that broke training.
- **Duplicate Edge Dedup:** `_resolve_all_inputs` deduplicates repeated input IDs while preserving order, so a canvas that emits two edges between the same two nodes no longer triggers spurious sibling-fan-in or double-merge behavior.
- **Redundant-Edge Suppression in Sibling Fan-In Warning:** `_merge_inputs` no longer warns when one input is a transitive ancestor of another (e.g. `Splitter` feeding both `Encoder` directly **and** via `Scaler`). That's a harmless redundant edge — the descendant supersedes under last-wins, so the merge is effectively just "use the descendant". Warning still fires for true sibling fan-in (independent branches sharing only a deeper common ancestor).

### ✨ Features
- **Sibling Fan-In Advisory (Bug 9j):** `_merge_inputs` detects when two or more inputs to the same node share a transitive ancestor (e.g. `Scaler` and `Encoder` both forked off the same `Splitter`) and records a `sibling_fan_in` advisory. Warnings flow through `PipelineExecutionResult.merge_warnings` → `PreviewResponse.merge_warnings` so the UI can surface a "column union + last-wins" hint instead of users guessing why one branch's columns silently overwrite the other's.
- **Merge Warning Detail Fields:** Engine now reports `overlap_columns` (the actual columns subject to last-wins overwrite) and `winner_input` (which input's values won) on every `sibling_fan_in` advisory, so the UI can name the concrete columns at risk and the branch that "won" instead of generic phrasing.
- **Collapsible Merge Advisory Banner:** `ResultsPanel` collapses merge warnings to a single-line summary by default (`▶ ⚠ N merge advisories — Node A, Node B`). Click to expand for full per-warning detail (inputs, overlap columns, winner, fix hint). Reduces visual noise on canvases with multiple advisories while keeping them one click away.
- **Merge Badge Severity Coloring:** `CustomNodeWrapper` merge badge turns amber with a ring when the node has an active `sibling_fan_in` advisory with `overlap_columns`; stays blue otherwise. Makes risky fan-in visible at a glance on the canvas.
- **Canvas Badge Legend:** New `?` button in the toolbar opens a popover explaining the three node-badge states (blue safe merge, amber risky merge, amber parallel) so users don't have to guess what the canvas is signaling. Expanded to cover success/failed status badges and all edge states (standard, branch-colored, shared dashed, winning merge edge).
- **Configurable Merge Strategy:** Nodes with 2+ distinct upstream sources now expose a **Merge Strategy** dropdown in the Properties panel (`last_wins` default, `first_wins` new). Engine reads `_merge_strategy` from node params and swaps column-overlap resolution accordingly. The `sibling_fan_in` advisory now carries the active `strategy` and picks the correct `winner_input` (last vs first).
- **Winner-Edge Highlight:** After a preview run, the edge from each merge's `winner_input` to the merging node is rendered in amber with a thicker stroke and a "WINS MERGE" label, so users can see at a glance which branch's values actually survived the tiebreak. Computed in `FlowCanvas` from `merge_warnings` and rendered by `CustomEdge`.
- **Auto-Fix "Chain instead" Button:** Each expanded merge advisory row with `overlap_columns` gets a Wand2-icon button that rewires the fan-in into a linear chain via the new `chainSiblings` store action. Keeps the head sibling's upstream edges, strips the tail siblings' incoming edges, and re-routes through registry handles so the resulting graph is a clean linear pipeline.
- **Row-Concat Drop Advisory (Audit #5):** When `_merge_frames` falls back to row-wise concatenation across inputs with mismatched column sets, it now emits a `row_concat_drop` advisory in `merge_warnings` (with `dropped_columns` + `kept_columns`) instead of leaving the column loss as a log line only. `ResultsPanel` renders the new variant inline in the existing merge banner.
- **Honest Merge Badge (Audit #2):** `CustomNodeWrapper` no longer shows the blue Merge badge on nodes whose registry definition declares no input ports (i.e. dataset/data-loader nodes). Multi-edge wiring on a non-mergeable node was previously visible as "merging N sources" even though the engine ignored the extras.
- **Pre-Flight Multi-Input Lint (Audit #7):** `useGraphStore.onConnect` now confirms every multi-input wiring on non-training nodes (the training/tuning prompt was already there). The dialog explains the column-union + last-wins contract before the user accepts the connection, so the merge contract is visible at wire time, not just after a preview run.
- **`pack_pipeline_output` Diagnoses Lost `y` (Bug 9d):** When a caller passes `was_tuple=True` but `y is None`, `skyulf-core/skyulf/utils.py` now logs a warning instead of silently flattening the shape. Surfaces upstream wiring bugs the moment the tuple shape is lost.
