# Changelog — 0.3.x Series

## v0.3.0 — Multi-Path Canvas Execution

### 🔧 Backend
- **Multi-Input Merge:** Training nodes (`basic_training`, `advanced_tuning`) now consume ALL upstream inputs instead of only the first. Added `_resolve_all_inputs()`, `_merge_inputs()`, and `_to_dataframe()` helpers to `PipelineEngine`.
- **Auto-Merge Strategy:** Column-wise concat when row counts match (parallel preprocessing), row-wise concat when columns match (data augmentation). Duplicate columns are automatically deduplicated.
- **Merge Conflict Detection:** Clear error messages when inputs are incompatible (no common columns, ambiguous shapes).
- **Topological Input Ordering:** Inputs merged in deterministic order based on pipeline topology.

### 🎨 Frontend
- **Merge Indicator Badge:** Nodes with 2+ incoming edges show a merge icon (⊕) with the input count in the header.
- **Connection Validation:** Model-to-model connections (training → training) are now blocked with a descriptive alert.

### 🐛 Bug Fixes
- **Dead-End Branch Pruning:** Nodes connected to the dataset but not on a path to any terminal node (training/preview) are now excluded from execution. Previously, dead-end branches (e.g., a Drop Rows node connected to Dataset but not wired downstream) would still execute and pollute the transformer chain.
