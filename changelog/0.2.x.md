# Changelog — 0.2.x Series

All notable changes for the **0.2.x** release series are documented here.

For the version index and summary, see the root [CHANGELOG.md](../CHANGELOG.md).

---

## v0.2.0
**"The Data Drift Redesign"**

### 📊 Data Drift Page
- **Enriched Job Metadata:** Backend `/monitoring/jobs` endpoint now cross-references filesystem artifacts with DB records to return `model_type`, `target_column`, `n_features`, `n_rows`, `description`, and `best_metric` per job.
- **Date-Grouped Dropdown:** Job selector dropdown now groups jobs by date with sticky date headers, showing model type badges, best metric, target column, and descriptions inline.
- **Job Descriptions:** New PATCH `/monitoring/jobs/{job_id}/description` endpoint allows saving descriptions to training jobs via the existing `job_metadata` JSON column — no migration needed.
- **Inline Description Editing:** Selected job shows a metadata bar with editable description (pencil icon, inline input, Enter to save, Escape to cancel).
- **Searchable Metadata:** Dropdown search now filters across dataset name, job ID, description, model type, and target column.

### � Drift History & Timeline
- **Drift Check Persistence:** New `DriftCheckResult` DB model stores every drift calculation (job_id, dataset_name, per-column PSI/Wasserstein/KS summary, full column_drifts JSON, row counts, timestamps).
- **History Endpoint:** New `GET /monitoring/drift/history/{job_id}` returns all past drift checks for a job, ordered by most recent.
- **Timeline Chart:** When 2+ drift checks exist for a job, a LineChart renders drifted-column count and drift % over time.

### 🎯 Feature Importance × Drift Risk
- **Feature Importance Extraction:** Training engine now extracts feature importances from fitted models (tree-based `.feature_importances_` and linear `.coef_`) and saves them in the metrics dict for both basic training and advanced tuning.
- **Enriched Drift Response:** `POST /drift/calculate` now returns `EnrichedDriftReport` with `feature_importances` overlay loaded from the training job.
- **Risk Column:** Drift table shows a Risk badge per feature — High (top-5 importance + drifted), Medium (top-15 + drifted), or Low — with an importance bar visualization.

### �🔧 Backend
- **Data Shape Persistence:** Training engine now saves `n_rows` and `n_features` to the metrics dict at the end of both basic training and advanced tuning, so monitoring can display dataset dimensions.
- **Monitoring Router Type Safety:** All SQLAlchemy column accesses use `typing.cast` for clean mypy compliance.

### 📈 Data Drift — Quick Wins
- **Summary Cards:** 4 stat cards at the top of the drift report (Total Columns, Drifted Count, Avg PSI with severity label, Most Drifted feature).
- **Show Only Drifted Filter:** Toggle button to hide stable columns; empty-state message when no drifted columns exist.
- **Table Sorting:** All metric columns are sortable (asc → desc → clear cycle) with sort icons.
- **Dropdown Metric Overflow:** Job dropdown shows only the first metric; full string on hover.
- **Empty State Placeholder:** Activity icon + guidance text when no drift report exists.
- **Schema Drift Improvements:** Collapsible Missing/Extra Columns sections with chevron animation, count badges, and explanatory subtitles.
- **Performance:** `groupedJobs` and `selectedJobData` wrapped in `useMemo`; metric cells use `tabular-nums` and null-safe `toFixed`.

### 🚀 Data Drift — Medium Impact
- **Export Drift Report (CSV):** Download button in the filter bar exports all columns, metrics, and risk levels to a CSV file (pure client-side).
- **Custom Drift Thresholds:** Collapsible settings panel with numeric inputs for PSI, KS p-value, Wasserstein, and KL Divergence thresholds. Values are passed to the backend `DriftCalculator`. Reset-to-defaults button included.
- **Sidebar Alert Badge:** Red dot badge on the "Data Drift" nav item when any recent drift check found significant drift. New lightweight `GET /monitoring/drift/status` endpoint returns `has_drift`, `drifted_jobs`, and `latest_check`.

### ✨ Data Drift — Nice-to-Have
- **Per-Column Historical Sparklines:** Tiny inline SVG sparklines in each table row showing PSI trend across past drift checks. Color-coded green/amber/red based on latest value. Only shown when 2+ drift checks exist.
- **Client-Side Threshold Re-evaluation:** Changing PSI/KS/Wasserstein/KL thresholds instantly re-classifies all columns as drifted/stable without re-uploading data. Summary cards, filter counts, table, and CSV export all reflect live threshold values.
