# Changelog — 0.7.x Series

## v0.7.0

Kicks off the 0.7.x series with the first model-explainability feature: a SHAP Explainability tab in the Experiments page with four sub-views — a cross-run Summary bar chart (mean |SHAP value| per feature), plus per-run Beeswarm, Dependence, and Waterfall plots for digging into individual predictions, similar to the graph types the `shap` library itself provides. Backend/skyulf-core gain a new optional `shap` dependency and a lightweight, training-time explainability computation (`compute_shap_explanation`) that never blocks or fails a training job if SHAP can't run for a given model/dataset combination.

Adds two more sub-views to round out the SHAP tab: **Force Plot** (a compact, single-row version of Waterfall — same per-row `base_value`/`shap_values` data, no new computation needed) and **Interaction Plot** (a heatmap of the top feature-pairs' joint influence on predictions, via `shap.TreeExplainer.shap_interaction_values` — tree-based models only, e.g. Random Forest/Gradient Boosting/XGBoost; gracefully omitted for linear/kernel models). `compute_shap_explanation` now returns an additional `interactions` key (`{feature_names, matrix} | None`).

Also fixes chart readability in dark mode across the Dashboard, EDA page, and the new SHAP views: chart colors are now centralized in a single reactive theme module (`core/theme/chartTheme.ts` + `useChartTheme()` hook) instead of each component computing colors ad hoc, so charts update live when the theme is toggled rather than only at first render. This also fixed a real bug where pie chart tooltips (Dashboard, EDA overview) rendered black, unreadable text on a dark background — Recharts' default tooltip item/label styles default to black unless explicitly overridden, which is now handled centrally.

Adds **Segmentation** as a first-class, trainable model type (previously clustering only existed as a fixed, non-configurable KMeans(k=3) add-on inside EDA profiling). A new `kmeans` model (skyulf-core `modeling/clustering.py`) plugs into the existing training/evaluation pipeline with zero changes to the sklearn wrapper layer, using an empty `target_column` as the established "no target" sentinel. Evaluation gains clustering-specific metrics (silhouette, Calinski-Harabasz, Davies-Bouldin scores) and a cluster summary (sizes + per-cluster centroids/mean feature values) via a new `evaluate_clustering_model()`. The frontend gets a dedicated **Segmentation** canvas node (reuses the Basic Training node's model/hyperparameter UI, with the target-column selector and Cross-Validation section hidden since neither applies to unsupervised clustering) and a new **Segmentation** tab in the Experiments page showing cluster-quality score cards, a cluster-size bar chart, and per-cluster centroid cards. Advanced Tuning's model dropdown now excludes clustering algorithms, since tuning requires a supervised scoring metric that doesn't apply to unsupervised clustering.
