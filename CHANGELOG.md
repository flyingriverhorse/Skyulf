All notable changes to the Skyulf project are documented here, organized by release series.

Each major/minor series has its own file in the [`changelog/`](changelog/) folder.

Segmentation (clustering) as a first-class model type with 4 algorithms (K-Means, Mini-Batch K-Means, Gaussian Mixture, Birch) and auto-labeling (profile + reference-column crosstab) (v0.7.0); unifies Basic Training and Advanced Tuning into a single `training` step/`TrainingJob` flow end-to-end — orchestration, node UI, DB schema, legacy canvas nodes, and a fixed-mode ensemble training crash fix — plus a dedicated Ensemble category in the job-tracking UI (Job History, Jobs page, Experiments filters, Job Details badge) (v0.7.1); hard-blocks pipelines that fit data-dependent preprocessing before a train/test split (backend + canvas pre-flight check), fixes a solver/penalty validation gap that let invalid `LogisticRegression` configs reach sklearn's opaque error, normalizes the deprecated `penalty` param and bridges `ConvergenceWarning` into the job log across all sklearn models, and raises `max_iter` ceilings to reduce convergence issues (v0.7.1 continued); adds outlier-handling, feature-selection-strategy, feature-generation, time-series, and geo-feature showcases across the `skyulf-core` example notebooks (00/01/05/07/08), fixing 3 real library bugs found along the way — a Polars→pandas target-conversion gap and a silently no-op `UnivariateSelection` default in `feature_selection/_common.py`, plus an `EDAVisualizer` scatter-matrix color/NaN shape-mismatch crash — with matching README updates and a full zero-error, no-pandas re-verification pass (v0.7.1); fixes two more `StatefulEstimator` dataset-shape bugs (target leakage when `X`/`y` are passed separately but `X` still contains the target column, and `(X, y)` tuples with a DataFrame `y` misclassified as pre-split `(train_df, test_df)` pairs), removes a duplicate tuned-model artifact save, and hardens a Celery DB-engine init race, ingestion-status defaults, and stuck-"running" job-polling aggregate status (v0.7.1 continued); fixes an SSRF via unrestricted S3 connector `endpoint_url` and an unauthenticated `/health/detailed` info-disclosure (v0.7.1 continued); `skyulf-core` usability pass — top-level re-exports for EDA/evaluation/explainability helpers and `NodeRegistry` (plus new `list_transformers()`/`list_models()`), a clear `predict()` error when the target column is left in the input, registry error messages that keep their "Available: [...]" hint, and `viz`-extra install messages pointing at `skyulf-core[viz]` (v0.7.1 continued); two Ruff lint fixes (`isinstance` union syntax, unused import) (v0.7.1 continued) |
